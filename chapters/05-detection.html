F. What a loop means here

In this chapter we use boundary loop with the same definition given in Chapter 2. We do not introduce a new term here.

Interpretation depends on local geometry: where concepts sit, which edges appear at birth, and whether cycle edges cross community boundaries in that year.

Community boundary. A community boundary is the interface between concept groups produced by graph partitioning (Louvain in this project). Crossing this boundary means an edge links two semantic regions that are locally close but still partitioned.

G. From representative cycle to bridge candidate

  1. Identify a loop feature I_k=[b_k,d_k) and its birth edge e_b=(u,v).
  2. Find the shortest path P_{b_k^-}(u,v) in the graph just before birth.
  3. Form the representative cycle C_k = e_b ∪ P_{b_k^-}(u,v).
  4. Partition the separation graph into communities c_t(·).
  5. Extract cycle edges (i,j) ∈ C_k with c_t(i) ≠ c_t(j) as bridge candidates.

\( E_t^{bridge}=\{(i,j)\in C_k \mid c_t(i)\neq c_t(j)\} \)

This is the exact extraction rule used in this site. Type-wise, \(E_t^{bridge}\) is a set of concept-pair edges. The output is not a topic label.
