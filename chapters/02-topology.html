C. Core idea

Interpretive view. We track boundary loops in each year’s concept graph. A boundary loop is useful when it crosses semantic communities, because those cross-community edges are concrete candidate relations.

Operational view. For each year t, compute persistent H₁ on a Vietoris–Rips filtration. Reconstruct one representative cycle at loop birth. Keep cycle edges (u,v) only when c_t(u) ≠ c_t(v). The retained edges are bridge candidates for ranking and temporal validation.

Working definitions used in this site

  • Boundary loop: a persistent H₁ feature interpreted as an unresolved boundary pattern.
  • Bridge candidate: a cycle edge whose two endpoints have different community labels in the same year.
  • Representative cycle: one computable witness used to read and extract candidate edges.
Persistent homology (quick definition). Persistent homology tracks which topological features appear and disappear as the distance threshold grows. For H₁, the feature of interest is a loop-like hole. Persistence length indicates how long that hole remains unresolved across thresholds.
Representative cycle. A representative cycle is a readable witness for one loop feature. In this project it is reconstructed from a birth edge plus a pre-birth shortest path. It supports interpretation, but it is not the only valid witness for the same homology class.
